    - Test 1: Middleware calls next() immediately (does not block request processing)
    - Test 2: On res finish, logger.info is called with { method, path, status, durationMs, consumerId }
    - Test 3: consumerId comes from (req as any).auth?.clientId when present
    - Test 4: consumerId is "anonymous" when req.auth is undefined (e.g., health endpoint)
    - Test 5: req.body is NOT present in log output (even if req has a body)
    - Test 6: req.headers.authorization is NOT present in log output
    - Test 7: durationMs is a rounded number (not bigint, not float with 15 decimals)
  </behavior>
  <action>
    Create `src/middleware/` directory. Write `request-logger.test.ts` first with mock req/res/next objects following Express 5 types. Mock `logger` from `../logger.ts` using `mock.module`. Then implement `request-logger.ts`:

    - Export function `requestLogger(req: Request, res: Response, next: NextFunction): void`
    - Capture `start = process.hrtime.bigint()` at entry
    - Call `next()` immediately
    - On `res.on("finish", ...)` compute `durationMs = Number(process.hrtime.bigint() - start) / 1_000_000`
    - Call `logger.info("request", { method: req.method, path: req.path, status: res.statusCode, durationMs: Math.round(durationMs), consumerId: (req as any).auth?.clientId ?? "anonymous" })`
    - Log ONLY these 5 fields. No req.body, no req.headers, no res.body. This is a security requirement.

    Use Express types: `import type { Request, Response, NextFunction } from "express";`
  </action>
