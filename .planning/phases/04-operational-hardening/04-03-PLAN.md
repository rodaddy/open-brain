  <name>Task 1: GitHub Actions CI pipeline</name>
  <files>.github/workflows/ci.yml</files>
  <action>
    Create `.github/workflows/` directory. Write `ci.yml`:

    ```yaml
    name: CI
    on:
      push:
        branches: ["wip/**", "feat/**", "fix/**"]
      pull_request:
        branches: [main]

    jobs:
      check:
        runs-on: ubuntu-latest
        services:
          postgres:
            image: pgvector/pgvector:pg16
            env:
              POSTGRES_DB: open_brain_test
              POSTGRES_USER: test
              POSTGRES_PASSWORD: test
            options: >-
              --health-cmd pg_isready
              --health-interval 10s
              --health-timeout 5s
              --health-retries 5
            ports:
              - 5432:5432
        steps:
          - uses: actions/checkout@v4
          - uses: oven-sh/setup-bun@v2
            with:
              bun-version: "1.3.9"
          - run: bun install
          - name: Typecheck
            run: bunx tsc --noEmit
          - name: Unit tests
            run: bun test
    ```

    Key decisions per research:
    - `pgvector/pgvector:pg16` matches production Postgres 16 on 10.71.20.49
    - Bun pinned to 1.3.9 matching local dev
    - Branch patterns match project convention: wip/*, feat/*, fix/*
    - Health check gives 50s for Postgres to be ready (10s interval x 5 retries)
    - Service container env vars are test-only, not real credentials
    - No integration tests for now (stretch goal per research) -- unit tests with mocks already thorough
  </action>
