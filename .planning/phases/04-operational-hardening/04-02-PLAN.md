    Write `scripts/backfill.test.ts` first. Mock `../src/db/pool.ts` (createPool returns mock pool), `../src/embedding.ts` (generateEmbedding and contentHash), and `../src/logger.ts`. Use bun:test `mock.module`. Test the backfill function by exporting it (not just running at module level).

    Then implement `scripts/backfill.ts`:

    ```
    TABLES config array:
    - thoughts: textFn = (r) => r.content
    - decisions: textFn = (r) => `${r.title}\n${r.rationale}`
    - relationships: textFn = (r) => [r.person_name, r.context, r.notes].filter(Boolean).join("\n")
    - projects: textFn = (r) => [r.name, r.description].filter(Boolean).join("\n")
    - sessions: textFn = (r) => r.summary
    ```

    Export `async function backfill(pool: pg.Pool, embedFn: typeof generateEmbedding): Promise<{ processed: number; failed: number }>` for testability.

    Main logic:
    1. For each table, SELECT * FROM {table} WHERE embedding IS NULL
    2. Log `Backfill: {table}` with `{ nullCount: rows.length }`
    3. For each row, compute text via textFn, call embedFn
    4. On success: UPDATE {table} SET embedding = $1, content_hash = $2, embedded_at = NOW(), embedding_model = 'gemini-embedding-001' WHERE id = $3
    5. Use `toSql()` from `pgvector/pg` for the embedding value
    6. On failure (embedFn returns null): increment failed counter, continue
    7. Delay 150ms between rows: `await new Promise(r => setTimeout(r, 150))`
    8. After all tables, log final summary
    9. Call pool.end()

    Add `import.meta.main` guard at bottom that creates pool, calls backfill, and exits with code 0 on success or 1 on fatal error (pool creation failure, etc.).

    Add `"backfill": "bun run scripts/backfill.ts"` to package.json scripts.

    CRITICAL: The text construction for each table MUST match exactly what the tool handlers use. The TABLES config above is derived from direct analysis of each tool handler file.
