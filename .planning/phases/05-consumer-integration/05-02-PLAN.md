**1. Create `hooks/open-brain-session-save.ts` (PreCompact hook):**

The script must use `#!/usr/bin/env bun` shebang. It:
- Reads stdin JSON to get `session_id`, `cwd`, `trigger`, `custom_instructions`
- Derives project name from `cwd` (basename of the directory path)
- Reads OPEN_BRAIN_AGENT_TOKEN from environment; if missing, silently exits (exit 0)
- Performs two-step MCP handshake to Open Brain at http://10.71.20.49:3100/mcp:
  a. POST initialize request with Bearer auth header -> extract `mcp-session-id` from response headers
  b. POST tools/call for `session_save` with the mcp-session-id header
- The session_save arguments:
  - `summary`: "Auto-saved before {trigger} compaction. Session: {session_id}. {custom_instructions}" (include custom_instructions if non-empty)
  - `project`: derived from cwd basename
  - `tags`: ["auto-save", "pre-compact", trigger]
  - `next_steps`: [] (empty -- we don't have structured data from the hook)
  - `key_decisions`: [] (empty)
- Wraps everything in try/catch -- on ANY error, silently exit 0 (never block compaction)
- Keep the script under 80 lines

Reference the code example in 05-RESEARCH.md "PreCompact Hook" section, but enhance with custom_instructions handling.

**2. Create `hooks/open-brain-session-load.ts` (SessionStart hook):**

The script must use `#!/usr/bin/env bun` shebang. It:
- Reads stdin JSON to get `cwd` and `trigger`
- Derives project name from `cwd` (basename)
- Reads OPEN_BRAIN_AGENT_TOKEN from environment; if missing, silently exits
- Performs two-step MCP handshake:
  a. POST initialize -> extract mcp-session-id
  b. POST tools/call for `session_load` with arguments `{ "project": "<project>" }`
- Parses the response: result.result.content[0].text
- If no sessions found (content starts with "No sessions"), silently exit
- Otherwise, parse the JSON content and output a concise summary to stdout:
  ```
  <!-- Open Brain: Previous Session Context -->
  ## Last Session: {project}
  **Saved:** {created_at}

  ### Key Decisions
  - {each decision}

  ### Blockers
  - {each blocker}

  ### Next Steps
  - {each step}
  <!-- End Open Brain Context -->
  ```
- Only include sections that have data (skip empty arrays)
- Wrap in try/catch -- on error, silently exit 0 (never block session start)
- Keep under 100 lines
- MUST complete within 5 seconds (the server is on local network, should be fast)

Reference the code example in 05-RESEARCH.md "SessionStart Hook" section.

<task type="auto">
  <name>Task 2: Build n8n Discord thought capture workflow</name>
  <files></files>
  <action>
Build an n8n workflow that captures Discord messages and sends them to Open Brain as thoughts. Use `mcp2cli n8n` for all n8n operations.

**Workflow design:**

Name: "Open Brain - Discord Thought Capture"

Nodes:
1. **Webhook Trigger** -- Receives Discord messages (via existing Discord bot integration or a new webhook). Use the Discord Trigger node if available (the existing "Discord Interactions" workflow with 48 nodes suggests the Discord community node is installed).

   If Discord Trigger node is NOT available, use a Webhook node and configure the existing Discord bot to POST to the n8n webhook URL.

2. **Filter** (IF node) -- Filter to only process messages from designated channels or users. Check if message content is non-empty and from the correct source.

3. **HTTP Request** (Initialize MCP session) -- POST to `http://10.71.20.49:3100/mcp` with:
   - Headers: `Authorization: Bearer <N8N_TOKEN>`, `Content-Type: application/json`
   - Body: MCP initialize request JSON
   - Response: Extract `mcp-session-id` from response headers

   For the N8N_TOKEN: retrieve from vaultwarden:
   ```bash
   N8N_TOKEN=$(mcp2cli vaultwarden-secrets get_secret_fields --params '{"name":"Open Brain - Auth Tokens"}' 2>/dev/null | jq -r '.result.AUTH_TOKEN_N8N')
   ```
   Set this as an n8n credential or use it directly in the HTTP Request node header.

4. **HTTP Request** (Call log_thought) -- POST to `http://10.71.20.49:3100/mcp` with:
   - Headers: same Authorization + `mcp-session-id` from step 3
   - Body: `{ "jsonrpc": "2.0", "id": 2, "method": "tools/call", "params": { "name": "log_thought", "arguments": { "content": "{{$node['Discord Trigger'].json.content}}", "tags": ["discord", "capture"] } } }`

5. **Optional: Respond to Discord** -- Send a reaction or brief reply confirming capture.

**Alternative approach (if MCP session handshake is too complex in n8n):**

The research notes suggest considering a simple REST endpoint. If the two-step MCP handshake proves difficult in n8n's HTTP Request nodes (header extraction between chained requests), fall back to:
- Adding a `POST /api/thoughts` REST endpoint to Open Brain that accepts `{ content, tags }` directly
- This would require a small server code change (new route in src/index.ts or a new file)
- Only do this if the MCP handshake approach fails

**Steps to build:**

1. First, check what Discord nodes are available:
   ```bash
   mcp2cli n8n n8n_list_workflows --params '{}' | jq '.result[] | select(.name | test("[Dd]iscord"))'
   ```

2. Create the workflow:
   ```bash
   mcp2cli n8n n8n_create_workflow --params '{
     "name": "Open Brain - Discord Thought Capture",
     "nodes": [...],
     "connections": {...}
   }'
   ```

3. Activate the workflow:
   ```bash
   mcp2cli n8n n8n_activate_workflow --params '{"workflowId": "<id>"}'
   ```

4. Test with a manual webhook trigger or test execution:
   ```bash
   mcp2cli n8n n8n_test_workflow --params '{"workflowId": "<id>"}'
   ```

Note: The exact node configuration depends on what Discord nodes are installed in the n8n instance. The executor should inspect existing Discord workflows for reference patterns.
