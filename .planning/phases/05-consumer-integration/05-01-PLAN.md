<task type="auto">
  <name>Task 2: Register Open Brain in mcp2cli and create agent-reference.md</name>
  <files>.planning/agent-reference.md</files>
  <action>
1. Add Open Brain to mcp2cli services.json. Read the current file at `~/.config/mcp2cli/services.json`, add the open-brain entry following the HTTP backend pattern (matching vaultwarden-secrets):

   ```json
   "open-brain": {
     "backend": "http",
     "url": "http://10.71.20.49:3100/mcp",
     "headers": {
       "Authorization": "Bearer <AGENT_TOKEN_FROM_VAULTWARDEN>"
     }
   }
   ```

   For the Bearer token value, retrieve the agent token from vaultwarden:
   ```bash
   AGENT_TOKEN=$(mcp2cli vaultwarden-secrets get_secret_fields --params '{"name":"Open Brain - Auth Tokens"}' 2>/dev/null | jq -r '.result.AUTH_TOKEN_AGENT')
   ```

   Use the AGENT_TOKEN as the Bearer value in the services.json entry (mcp2cli uses the agent role for CLI access -- admin is too broad, readonly can't write).

2. Generate skill files:
   ```bash
   mcp2cli generate-skills open-brain
   ```
   This auto-generates `~/.config/mcp2cli/skills/open-brain/SKILL.md` by introspecting the live server's tool list.

   NOTE: This requires the Open Brain server to be running at 10.71.20.49:3100 with the agent token active. If the server isn't deployed with the new tokens yet, this step will fail -- handle in the checkpoint below.

3. Verify mcp2cli can list tools:
   ```bash
   mcp2cli open-brain --help
   ```

4. Verify a real tool call works:
   ```bash
   mcp2cli open-brain search_brain --params '{"query":"test"}'
   ```

5. Create `.planning/agent-reference.md` with non-secret reference data:

   - Token roles: admin, agent, discord, n8n, readonly
   - mcp2cli service name: open-brain
   - Port: 3100
   - Database: open_brain on 10.71.20.49:5432
