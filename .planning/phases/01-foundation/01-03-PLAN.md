<task type="auto">
  <name>Task 1: Create MCP server, transport dispatch, and Express application</name>
  <files>src/server.ts, src/transport.ts, src/index.ts</files>
  <action>
**src/server.ts:** Create the McpServer singleton factory.
- Import `McpServer` from `@modelcontextprotocol/sdk/server/mcp.js`
- Export `createBrainServer()` that returns a new `McpServer({ name: "open-brain", version: "1.0.0" })`
- This is intentionally minimal -- tool registration happens in Phase 2. The server just needs to exist and accept connections.

**src/transport.ts:** Session-aware transport dispatch.
- Import `StreamableHTTPServerTransport` from `@modelcontextprotocol/sdk/server/streamableHttp.js`
- Import `isInitializeRequest` from `@modelcontextprotocol/sdk/types.js`
- Import `randomUUID` from `node:crypto`
- Import type `McpServer` from `@modelcontextprotocol/sdk/server/mcp.js`
- Import type `Request, Response` from `express`
- Maintain a `transports: Record<string, StreamableHTTPServerTransport>` map
- Export `createTransportHandlers(server: McpServer)` that returns an object with three Express handlers:
  1. `handlePost(req, res)`: Check `mcp-session-id` header. If session exists, dispatch to existing transport. If no session and `isInitializeRequest(req.body)`, create new `StreamableHTTPServerTransport` with `sessionIdGenerator: () => randomUUID()`, wire `onsessioninitialized` to store in map, set `transport.onclose` to remove from map, call `server.connect(transport)`, then `transport.handleRequest(req, res, req.body)`. Otherwise return 400.
  2. `handleGet(req, res)`: Look up session by `mcp-session-id` header. If found, call `transport.handleRequest(req, res)`. If not found, return 400.
  3. `handleDelete(req, res)`: Look up session, call `transport.close()`, delete from map, return 200.

**src/index.ts:** The Express application entry point.
- Import `express` and `cors`
- Import `createPool`, `checkPoolHealth` from `./db/pool.ts`
- Import `buildTokenMap`, `authMiddleware` from `./auth.ts`
- Import `createBrainServer` from `./server.ts`
- Import `createTransportHandlers` from `./transport.ts`
- Import `runMigrations` from `./db/migrate.ts`
- Import `logger` from `./logger.ts`
- Create Express app with:
  1. `app.use(cors())` -- multi-consumer access
  2. `app.use(express.json())` -- JSON body parsing
  3. `GET /health` (NO auth) -- calls `checkPoolHealth(pool)`, fetches `${LITELLM_URL}/health` with 3s `AbortSignal.timeout`, returns `HealthStatus` JSON. 200 if both healthy, 503 if degraded.
  4. Build token map from `process.env`
  5. Create auth middleware from token map
  6. Apply auth middleware to `/mcp` routes only: `app.post("/mcp", auth, handlers.handlePost)`, `app.get("/mcp", auth, handlers.handleGet)`, `app.delete("/mcp", auth, handlers.handleDelete)`
- On startup:
  1. Create pool
  2. Run migrations (catch error, log, but don't crash -- migration may have already been applied)
  3. Create MCP server
  4. Create transport handlers
  5. Listen on `process.env.PORT || 3100`
  6. Log startup message with port

Export `createApp(pool, tokenMap)` as a separate function for testing (takes injected dependencies, returns Express app without calling listen). The `if (import.meta.main)` block handles actual startup.
